## Label Distribution Fix

**Problem**: Initial ATR thresholds (ATR_MULTIPLIER=1.0) created nearly all HOLD labels, preventing model training.

**Solution**: Lowered thresholds to create balanced distribution:
- ATR_MULTIPLIER: 0.15 (from 1.0)
- ATR_MIN_THRESHOLD: 0.0005 (from 0.002)
- TRAINING_MIN_PROFIT_THRESHOLD: 0.001 (from 0.005)

**Result**: Balanced BUY/HOLD/SELL distributions (e.g., SELL: 74, HOLD: 283, BUY: 131)

## v1.9.1 - Model Accuracy Fix

**Date**: April 11, 2026

### Problem

Models showed ~11% accuracy due to:
1. Walk-forward test set too small (20 samples)
2. Only RF model trained (NN/GB set to 0)
3. Model saved from last fold, not trained on all data

### Fix Applied

| File | Change |
|------|--------|
| `src/ai_model.py` | Increased test_size to 50, removed redundant saved_model tracking, final model trains on ALL data |

### Code Changes

```python
# OLD: 20 sample test sets + saved from last fold only
test_end = train_end + 20
saved_model = model  # from last fold

# NEW: 50 sample test sets + final model trained on all data
test_size = 50  # Larger test set
final_model.fit(X, y)  # Train on all data
saved_model = final_model
```

### Test Results (Expected After Retraining)

- Larger test sets (50 samples) → more stable accuracy
- Final model trained on all data → better production predictions
- Accuracy now reflects validation on larger sample size
