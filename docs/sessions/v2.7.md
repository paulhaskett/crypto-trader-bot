## Root Cause: LTC-GBP Not Selling

**Bug Location**: `src/trading_engine.py` lines 730-735 (original)

**Problem**:
- Close condition was: `if trailing_hit and current_price >= break_even`
- But `trailing_stop` is floored at `break_even` (line 715: `trailing_stop = max(raw_trailing, break_even)`)
- This means condition can ONLY be true if `price == break_even` exactly
- The `final_stop` variable was calculated but **never used** in the close condition

**Fix**:
- Changed close condition to: `if current_price <= final_stop`
- `final_stop` already incorporates trailing stop, initial stop floor (90%), and break_even protection

## Smart Scale-In Logic Implemented

**Before**: Scale-in executed immediately when conditions met (could drain GBP balance)

**After**:
1. Collect candidates into `scale_in_candidates[]` list
2. Calculate recovery score: `score = signal_confidence / max(drop_from_peak, 0.01)`
3. Add +50% bonus for bull/uptrend regimes
4. Filter by `SCALE_IN_MIN_SIGNAL_CONFIDENCE` (default 60%)
5. After position loop, sort by score descending
6. Execute ONLY the best candidate per cycle
7. Check budget: `SCALE_IN_MAX_GBP_PER_CYCLE` (default £15)

**Recovery Score Formula**:
```python
score = signal_confidence / max(drop_from_peak, 0.01)
if regime in ['bull', 'uptrend']:
    score *= 1.5
```
