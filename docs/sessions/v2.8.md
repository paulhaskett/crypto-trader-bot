## Bugs Fixed

### 1. Model Loading Bug (Critical)
**Problem**: `load_existing_*` methods stored keys with underscores (`BTC_GBP`) but `predict()` looked with hyphens (`BTC-GBP`).

**Fix**: Added conversion in `load_existing_*` methods:
```python
product_id_underscore = model_file.replace('_rf_model.joblib', '')
product_id = product_id_underscore.replace('_', '-')
self.models[product_id] = joblib.load(f)
```

### 2. Regime Detection Overhaul
**Problem**: Regime said "uptrend" when chart showed neutral/flat. RSI > 60 was being used as "strong uptrend" which is wrong (that's overbought).

**Fix**: Simplified to match what humans see on charts:
```python
if current_price > short_ma and current_price > long_ma:
    regime = 'uptrend'  # Price above both MAs
elif current_price < short_ma and current_price < long_ma:
    regime = 'downtrend'  # Price below both MAs
else:
    regime = 'neutral'  # Price between MAs
```

### 3. Training Data Increase
**Problem**: 60 days of training data was insufficient for accurate predictions.

**Fix**: Changed `collect_multi_source_data()` call from 60 → 180 days in `train_model()`.

### 4. f-String Syntax Errors
**Problem**: `detect_regime()` had malformed f-strings with `{short_period}` missing closing braces.

**Fix**: Rewrote logger calls to use `%` formatting:
```python
logger.info("Regime: %s = uptrend (price £%.2f > MA%d/MA%d, RSI: %.1f)" % (product_id, current_price, short_period, long_period, rsi))
```
