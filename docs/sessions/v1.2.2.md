## Problems Fixed

1. **Wrong displayed values**: Settings page showed placeholder defaults instead of actual config values
   - Root cause: Risk settings fetched from `/api/status` which doesn't include them
   - Fix: Created new `GET /api/settings/risk` endpoint

2. **Scale-In/Scale-Out showed off/empty**: Incorrect field names and value conversions
   - Scale-Out used wrong field (`percentages` instead of `take_profit_levels`)
   - Scale-In size was stored as decimal (0.25) but displayed as percentage (25)

3. **API 404 errors**: Scale-in/scale-out endpoints only existed in `main.py`
   - Root cause: API workers run from `src/api_worker.py`, not `main.py`
   - Fix: Added endpoints to `src/api_worker.py`

4. **JavaScript error**: `settings is not defined`
   - Root cause: Referenced Python `settings` object in JavaScript
   - Fix: Changed to hardcoded fallback value

## Key Understanding

### Multi-Process Architecture

The bot runs with two separate entry points:
- **Trading process**: `src/trading_loop.py`
- **API workers**: `src/api_worker.py` (via gunicorn)

All API endpoints used by the dashboard must be defined in `src/api_worker.py`, not just `main.py`.

### Take Profit vs Scale-Out TP Levels

The settings page shows two TP-related settings:
- **Take Profit** (1%): Sets `TAKE_PROFIT_LEVELS = [1.0, 2.0, 3.0]` (1R, 2R, 3R where R = risk)
- **Scale-Out TP1/2/3** (1%/2%/3%): Display only - shows calculated profit percentages

The Scale-Out TP levels on the settings page are informational, not controlling. Actual TP prices are calculated based on entry price and stop loss.

## v1.2.3 - Scale-In Fixes
**Date**: 2026-03-22

### Summary

Fixed scale-in configuration to allow more scale-ins to trigger.

### Problems Fixed

1. **Cooldown too long**: Scale-in cooldown was 6 hours, preventing multiple scale-ins
2. **Triggered levels not resetting**: Once levels were marked as "triggered", they stayed triggered even after cooldown expired

### Changes Made

#### 1. `config/settings.py`
Changed `SCALE_IN_COOLDOWN_HOURS` from 6 to 1:
```python
SCALE_IN_COOLDOWN_HOURS: int = 1  # Minimum hours between scale-ins (was 6)
```

#### 2. `src/trading_engine.py`
Added logic to reset triggered levels when cooldown expires AND price has recovered:
```python
# If cooldown expired, reset triggered levels so they can trigger again
if cooldown_ok and triggered_levels:
    # Check if price has recovered above the lowest triggered level
    lowest_triggered_level = min(triggered_levels)
    recovery_price = holding_entry * (1 - lowest_triggered_level / 100)
    if current_p > recovery_price:
        logger.info(f"Scale-in: {product_id} recovered, resetting triggered levels")
        triggered_levels = []
```

### Files Modified

| File | Changes |
|------|---------|
| `config/settings.py` | Reduced cooldown from 6 to 1 hour |
| `src/trading_engine.py` | Added triggered levels reset logic |
