# Session: v1.5.0 - Trading Parameters & Portfolio Protection
**Date**: April 4, 2026

## Changes Made

### 1. Trading Parameters (config/settings.py)

| Parameter | Old Value | New Value | Purpose |
|------------|-----------|-----------|---------|
| `MAX_DAILY_TRADES` | 3 | 2 | More selective daily trades |
| `MAX_CONCURRENT_POSITIONS` | 8 | 8 | One position per pair (reverted from 4) |
| `MAX_POSITION_SIZE` | 5% | 10% | Larger positions, fewer fees |
| `MIN_PROFIT_NEUTRAL` | 3% | 5% | Higher barrier in neutral |
| `MIN_PROFIT_BULL` | 5% | 8% | Let winners run more |
| `MODEL_CONFIDENCE_THRESHOLD` | 85% | 85% | More selective (reverted from 80%) |
| `PREDICTION_HORIZON` | 8h | 12h | Longer-term predictions |
| `MARKET_CHECK_INTERVAL` | 2700s (45min) | 3600s (60min) | Fewer cycles |
| `REPLACEMENT_CONFIDENCE_THRESHOLD` | 15% | 8% | Easier to replace weak positions |
| `REPLACEMENT_MIN_PROFIT` | (new) | 2% | Only replace winning positions |
| `REPLACEMENT_COOLDOWN_HOURS` | (new) | 24h | Don't replace same position quickly |
| `REPLACEMENT_MAX_PER_CYCLE` | (new) | 1 | Max 1 replacement per cycle |
| `TRADE_COOLDOWN_SECONDS` | (new) | 300s (5min) | Prevent rapid duplicate trades |
| `STOP_LOSS_MIN_PERCENT` | (new) | 10% (5-20% configurable) | Fixed stop loss |
| `REGIME_SHORT_MA_PERIOD` | 8h | 12h | Align with prediction horizon |
| `REGIME_LONG_MA_PERIOD` | 24h | 48h | 4x short for clear trend |
| `REGIME_THRESHOLD` | 0.5% | 1% | More stable regime detection |

### 2. Enhanced Replacement Logic (src/trading_engine.py)

- Only replaces positions with ≥2% profit
- Requires 8% confidence improvement to replace
- 24-hour cooldown between replacements
- Maximum 1 replacement per cycle
- Prevents replacing losing or small-profit positions

### 3. Trade Cooldown (src/trading_engine.py)

- 5-minute cooldown between trades for same product
- Prevents duplicate trades within short timeframe

### 4. Wallet Tracking (src/risk_manager.py)

- Logs wallet snapshot to `logs/wallet_snapshots.log` on startup
- Checks balance every 6 hours
- Warns if GBP drops >5% unexpectedly (logs only, no trading pause)

### 5. Settings Page (src/templates/settings.html)

- Stop loss now configurable: 5-20% (default 10%)
- Updated input range and labels

### 6. Take Profit Levels (src/risk_manager.py)

Updated TP levels to match new profit thresholds:

| Regime | Old TP Levels | New TP Levels |
|--------|---------------|---------------|
| Downtrend | 2%, 4%, 6% | 2%, 4%, 6% |
| Neutral | 4%, 6%, 8% | 5%, 7.5%, 10% |
| Uptrend | 5%, 7.5%, 10% | 8%, 12%, 16% |

## API Endpoints Added/Modified

None - all changes are internal logic and settings.

# Session: v1.6.0 - GBP Direct Training & Regime Features
**Date**: April 9, 2026

## Summary

Major architectural change: AI models now train directly on GBP pairs instead of USD pairs.

## Changes Made

### 1. GBP Direct Training (config/settings.py)

| Setting | Old Value | New Value | Purpose |
|---------|-----------|-----------|---------|
| `TRAINING_PAIRS` | USD pairs | GBP pairs | Direct market dynamics |

**Before**: Train on USD (BTC-USD, ETH-USD, etc.) → Apply to GBP trading
**After**: Train on GBP (BTC-GBP, ETH-GBP, etc.) → Direct trading

### 2. Regime Features Added (src/ai_model.py)

- Added `_add_regime_features()` method to generate one-hot encoded regime features
- Added regime features to model training: `regime_is_uptrend`, `regime_is_downtrend`, `regime_is_neutral`
- Added regime features to prediction: `_calculate_regime_feature()` in data_collector.py

### 3. Feature Count Fixed (src/ai_model.py, src/data_collector.py)

- Training: 20 features (including 3 regime features)
- Prediction: Now matches 20 features (was 17, causing scale errors)

### 4. Scale-in Bug Fixed (src/trading_engine.py)

- Added missing `'side': 'buy'` field before saving position to database
- Fixed `NOT NULL constraint failed: open_positions.side` error

### 5. Multi-Source Data Collection (src/data_collector.py)

- Added `collect_multi_source_data()` to combine Coinbase + CoinGecko data
- Falls back to Coinbase-only if CoinGecko unavailable

### 6. Training Data Reduced (src/ai_model.py)

- Changed from 365 days → 90 days (Coinbase now limits to 350 candles anyway)

## Files Modified

| File | Changes |
|------|---------|
| `config/settings.py` | TRAINING_PAIRS now uses GBP pairs |
| `src/ai_model.py` | Added regime features, fixed feature count, simplified model loading |
| `src/data_collector.py` | Added `_calculate_regime_feature()`, `collect_multi_source_data()` |
| `src/trading_engine.py` | Fixed scale-in side bug |

## Model Files

- Old USD models backed up to: `models/backup_usd_models/`
- Scalers deleted: `rm models/*_scaler.pkl` (need retraining with new features)

## Why This Change?

1. **Old reason for USD**: More historical data (365+ days vs 30-60 days)
2. **New reality**: Coinbase API now caps at 350 candles regardless of date range
3. **Benefit**: Direct GBP training captures exact market dynamics we trade in
