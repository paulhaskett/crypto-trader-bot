# Session: v5.0 - Settings Page Fix: Persistence & Correct Defaults

**Date**: May 31, 2026

## Summary

Fixed settings page to correctly display current configuration and persist changes across container restarts and gunicorn workers.

## Problems Fixed

### 1. Missing Settings Attributes Crashed API Endpoints

**Root Cause**: `settings.py` was missing `TAKE_PROFIT_LEVELS` and `MAX_SCALE_OUT_PER_POSITION` — they were dynamically set during POST but didn't exist on fresh startup. Both `GET /api/settings/risk` and `GET /api/scale_out/status` crashed with `AttributeError`.

**Fix** (`config/settings.py`): Added defaults:
```python
TAKE_PROFIT_LEVELS: list = [1.0, 2.0, 3.0]
MAX_SCALE_OUT_PER_POSITION: int = 3
```

### 2. Wrong Hardcoded Defaults in Frontend

**Root Cause**: `settings.html` had fallback values that didn't match `settings.py`.
| Setting | Before (wrong) | After (matches settings.py) |
|---------|---------------|---------------------------|
| Confidence threshold | 0.70 (70%) | 0.65 (65%) |
| Stop loss | 0.05 (5%) | 0.10 (10%) |
| Max position size | 0.05 (5%) | 0.45 (45%) |
| Max concurrent positions | 8 | 6 |
| Market check interval | 2700 (45min) | 14400 (4h) |

**Fix** (`settings.html`): Updated all 5 fallback values to match `settings.py`.

### 3. Scale-Out Showed Enabled When Disabled

**Root Cause**: `settings.html:387` checked `scaleOut.enabled !== false`. When API errored, `enabled` was `undefined`, and `undefined !== false` → `true`.

**Fix** (`settings.html`): Changed `!== false` to `=== true`.

### 4. Settings Not Persisted Across Workers/Restarts

**Root Cause**: `load_settings()` imported the Settings singleton — never loaded DB overrides. Post-saves only affected the current worker's in-memory copy.

**Fix**: Added `Settings.load_from_db()` method and call it from `load_settings()` on first invocation:

```
load_settings() first call
  → calls settings.load_from_db()
    → reads DB for: scale_in_enabled, scale_out_enabled, market_check_interval,
      model_confidence_threshold, stop_loss_min_percent, take_profit_level,
      max_position_size, max_concurrent_positions, display_currency
    → overrides Settings object attributes
    → logs "Loaded N settings from database"
  → caches loaded state
  → returns settings
```
