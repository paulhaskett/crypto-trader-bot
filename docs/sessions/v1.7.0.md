# Session: v1.7.0 - USD Training Models & Dashboard Fixes
**Date**: April 9, 2026

## Overview

Fixed the dashboard showing 0% confidence for all trading pairs after models were trained on USD pairs but the GBP trading pair mapping wasn't working correctly.

## Problem

After training models on USD pairs (for better accuracy), the dashboard showed:
- Models: 0/8 trained (should be 5-8 trained)
- Confidence: 0% for all pairs (should be 50-90%)
- ADA, LINK, UNI showed no signals despite having model files

## Root Causes

### 1. gbp_to_usd_map Not Working
The container's `ai_model.py` had outdated code that didn't build the GBP→USD mapping correctly:
- Container had: `training_pairs = settings.TRAINING_PAIRS` returning GBP pairs
- Local had: `training_pairs = settings.TRAINING_PAIRS` returning USD pairs

### 2. Missing Model Files
Only 5 of 8 USD model files existed in the container (BTC, ETH, SOL, LTC, DOT). The ADA, LINK, UNI models were missing despite existing on the host.

### 3. API Worker Model Check
The model status check in `api_worker.py` didn't include checking USD models when determining if a GBP pair was "trained".

### 4. Signal Cache Not Updating
The background scheduler that should refresh signals every 10 minutes wasn't running (jobs list was always empty).

# Before:

## Key Learnings

1. **USD training works**: Models trained on USD pairs achieve 50-77% accuracy vs 35-60% on GBP pairs
2. **Docker build caching**: Container rebuilds can use cached layers if build doesn't invalidate them
3. **Manual model sync**: Sometimes need to manually copy model files when Docker volumes aren't working correctly
4. **Scheduler issues**: APScheduler jobs list can be empty if jobs are added but scheduler isn't properly started
