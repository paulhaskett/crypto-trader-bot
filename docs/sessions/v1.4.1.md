## Regime Logic

| Regime | SELL Threshold | Scale-Out | Scale-In Levels |
|--------|---------------|-----------|-----------------|
| **Downtrend (Bear)** | 0.5% | 0.5% | 1%, 2%, 3% drops |
| **Neutral** | 1.0% | 1.0% | 2%, 4%, 6% drops |
| **Uptrend (Bull)** | 1.5% | 1.5% | 3%, 5%, 7% drops |

**Logic**: 
- Downtrend = take profit sooner, average down faster (tighter scale-in)
- Uptrend = hold for more profit, less aggressive with scale-ins (wider levels)

## Bug Fixes (March 26, 2026)

### Fix 1: Duplicate Trade Prevention
**Problem**: Scale-in trades saved twice with different order_ids:
- First save: `scale_in_{product_id}_{timestamp}` in `execute_signal()`
- Second save: `sdk_order_*` in `execute_live_trade()`

**Fix**: Commented out duplicate save in `trading_engine.py` line ~1619-1633. Now only saves in `execute_live_trade()` with real Coinbase order_id.

### Fix 2: Noisy 400 Error Logs
**Problem**: Coinbase SDK returned 400 errors when fetching order details after successful order placement. These were logged as WARNING, causing log spam.

**Fix**: Changed 400 error logging to DEBUG level in `coinbase_api.py` lines ~213-234.

### Fix 3: Confidence > 100%
**Problem**: Unanimous agreement 15% boost had no cap, causing confidence to show 106.9%.

**Fix**: Added `min(confidence * 1.15, 1.0)` cap in `ai_model.py` line ~654.

## Fix 4: Portfolio Value and PnL Calculations (March 26, 2026)

**Problem**: Portfolio value was showing ~£63 instead of ~£158, and PnL was showing ~£0.24 instead of ~£0.94.

**Root cause**: `api_worker.py` was using:
1. `remaining_size` instead of `size` for portfolio value (missing 70% of holdings)
2. `entry_price` instead of `weighted_entry_price` for PnL (not accounting for scale-ins)

**Fix**: Changed in `api_worker.py` lines 353-360 and 507-515:
```python
# Before (buggy):
size = pos.get('remaining_size', pos.get('size', 0))
entry_price = pos.get('entry_price', 0)

# After (fixed):
size = pos.get('size', 0)  # Use full size
entry_price = pos.get('weighted_entry_price', pos.get('entry_price', 0))  # Use weighted avg
```

**Result**: Portfolio now shows correctly (~£158), PnL shows correctly (~£1.34)
