## How the Trading System is Meant to Work

### Fee System

**Fee Fetching**:
- Fetches from Coinbase API via `get_transaction_summary()`
- SDK returns `GetTransactionSummaryResponse` object, NOT dict
- Access fee_tier as attribute: `response.fee_tier.get('maker_fee_rate')`
- Falls back to hardcoded defaults only if API fails
- Cached in database `user_settings` table

**Fee Rates**:
- Maker: 0.35% (Intro 2 tier)
- Taker: 0.75% (Intro 2 tier)
- Total: 1.10% (was using fallback 1.80% = WRONG!)

**Fee Impact**:
- On £15 trade: £0.27 at 1.8% vs £0.16 at 1.1%
- Positions need higher price to break even with wrong fees

---

### Position Monitoring (`monitor_positions()`)

The `monitor_positions()` function in `trading_engine.py` runs every trading cycle and checks if positions should be closed. The logic:

```
For each open position:
  1. Get current price from multi-source pricer
  2. Calculate break-even = entry_price × (1 + taker_fee)
  3. Track peak_price (always update if current > peak)
  4. Calculate trailing_activated = (peak_price >= break_even)
  5. Calculate trailing_stop = peak_price × (1 - trailing_pct)
     - Floor at entry_price × 0.95 (never sell below 95% of entry)
  6. Check AI SELL signal (NEW in v2.9.0)
  7. Check trailing stop conditions
  8. If should_close → execute sell order + update DB
```

**Key Variables**:
| Variable | Purpose |
|----------|---------|
| `entry_price` | Price at which position was opened |
| `break_even` | Entry + fees (price needed to cover costs) |
| `peak_price` | Highest price since position opened |
| `trailing_activated` | True when peak has been above break-even |
| `trailing_stop` | Dynamic stop that moves up with price |
| `trailing_pct` | 2% default (from `TRAILING_STOP_REGIME_MAP`) |

---

### Exit Conditions (in order checked)

1. **AI SELL Signal** (NEW v2.9.0):
   ```
   if signal['action'] == 'SELL':
       if current_price >= break_even OR profit >= 1%:
           close position
   ```

2. **Trailing Stop**:
   ```
   if trailing_activated AND current_price <= trailing_stop:
       close position
   ```

3. **Emergency Stop** (below break-even):
   ```
   if (entry - current) / entry > 2%:
       close position (emergency stop)
   ```

---

### Stop Loss Direction

**CRITICAL**: For long positions (buy), stop loss must be BELOW entry:
- Entry: £100
- Stop Loss: £95 (5% below)
- Break-even: £100.75 (covers fees)

The old code had stop_loss_price ABOVE entry (£1885 for ETH entry £1711), which acted as a take profit, not a stop loss!

## Common Bugs to Avoid

### 1. `trailing_activated` undefined
**Symptom**: NameError when monitoring positions
**Cause**: Variable referenced but never defined
**Fix**: Add `trailing_activated = peak_price >= break_even`

### 2. Fee parsing failure
**Symptom**: Using fallback fees (1.8%) instead of actual (1.1%)
**Cause**: `response.get('fee_tier')` fails on SDK object
**Fix**: Use `response.fee_tier.get('maker_fee_rate')` (access as attribute)

### 3. Stop loss wrong direction
**Symptom**: Position never stops (stop above entry for long)
**Cause**: Stop calculated as entry × 1.05 instead of 0.95
**Fix**: For long positions: `stop_loss = entry × 0.95`

### 4. AI SELL not checked for existing positions
**Symptom**: Positions stay open despite SELL signal
**Cause**: `monitor_positions()` only checked trailing stop, not AI signals
**Fix**: Add signal check before trailing stop evaluation

## v2.9.3 - Real-Time Peak Tracking

**Date**: May 10, 2026

### Problem

Peak price was only updating during the 4-hour trading cycle, meaning between cycles the peak in the database could be stale while the current price moved higher.

### Solution

Added real-time peak tracking to `get_all_open_positions_detailed()` in `database.py`:
- Whenever positions are fetched (dashboard, API), compare current price to stored peak
- If current > peak, update the peak in the database immediately
- This ensures peak is always current without waiting for trading cycle

### Code Change

```python
# In get_all_open_positions_detailed():
# Update peak price if current price is higher (track peak in real-time)
current_peak = pos.peak_price if pos.peak_price else pos.entry_price
if current_price > current_peak:
    pos.peak_price = current_price
    session.commit()
    logger.info(f"[REAL_TIME_PEAK] {product_id}: £{current_peak:.2f} -> £{current_price:.2f}")
```

### Result

Peak now updates on every dashboard/API access, not just during trading cycle.

## v2.9.7 - Loss-Selling Bug Fix & Configuration Updates

**Date**: May 13, 2026

### Problems Identified

Recent trades sold at a loss:
| Pair | Entry | Exit | Loss | Reason |
|------|-------|------|------|--------|
| SOL-GBP | £70.14 | £68.04 | -2.93% | Emergency stop (2% drop from entry) |
| ADA-GBP | £0.20 | £0.20 | -2.44% | Emergency stop (trailing_stop = entry) |
| LINK-GBP | £7.62 | £7.55 | -0.92% | Trailing stop hit (old activation logic) |

### Root Causes

1. **Emergency stop** (trading_engine.py line 796-799): Triggered when price dropped 2% from entry, **even if below break-even** (at a loss)
2. **Trailing stop floor**: When peak never went above entry, floor was set to entry * 0.95, still triggering at/near entry price
3. **Trailing activation**: Used old logic (at break-even) instead of 2% above break-even

### Changes Made

#### 1. Trailing Activation Buffer (v2.9.7a)
**Added**: `TRAILING_ACTIVATION_BUFFER` setting in `config/settings.py`

```python
TRAILING_ACTIVATION_BUFFER: float = 0.02  # 2% above break-even to activate trailing
```

Updated in both files:
- `src/trading_engine.py` (line 760): `activation_threshold = break_even * (1 + settings.TRAILING_ACTIVATION_BUFFER)`
- `src/api_worker.py` (line 942): Same logic for dashboard display

#### 2. Confidence Threshold Raised (v2.9.7b)
**Changed**: `MODEL_CONFIDENCE_THRESHOLD` from 50% to 65%

```python
MODEL_CONFIDENCE_THRESHOLD: float = 0.65  # 65% - reduce signals
```

#### 3. Emergency Stop Fix (v2.9.7c)
**Fixed**: Emergency stop now ONLY triggers if already in profit (above break-even)

```python
# OLD (WRONG - sold at loss):
elif (entry_price - current_price) / entry_price > 0.02:
    should_close = True

# NEW (CORRECT - only sell if was in profit):
elif current_price >= break_even and (entry_price - current_price) / entry_price > 0.02:
    should_close = True
```

#### 4. Trailing Stop Floor Fix (v2.9.7d)
**Fixed**: Floor now below break-even, not 95% of entry

```python
# OLD (WRONG):
trailing_stop = max(trailing_stop, entry_price * 0.95)

# NEW (CORRECT):
stop_floor = break_even * (1 - trailing_pct)
trailing_stop = max(trailing_stop, stop_floor)
```

This ensures:
- When price is between entry and break-even: trailing stop is below break-even (won't trigger)
- When price is above break-even: trailing stop activates normally

|------|---------|
| `config/settings.py` | Added `TRAILING_ACTIVATION_BUFFER: 0.02`, changed `MODEL_CONFIDENCE_THRESHOLD: 0.65` |

| Scenario | Before | After |
|----------|--------|-------|
