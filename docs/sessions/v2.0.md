## Volatility Regime Filter

### How It Works

| vol_ratio | Regime | ATR Multiplier (k) |
|-----------|--------|-------------------|
| > 1.5 | high | 0.25 |
| 0.7 - 1.5 | normal | 0.15 |
| < 0.7 | low | 0.10 |

### Detection Formula
```python
vol_ratio = ATR_short(24) / ATR_long(72)
if vol_ratio > 1.5: regime = 'high', k = 0.25
elif vol_ratio < 0.7: regime = 'low', k = 0.10
else: regime = 'normal', k = 0.15
```

### Implementation
- `detect_volatility_regime()` method in ai_model.py
- Integrated into `create_target_labels()` for training
- Added to `predict()` output for real-time signals

**Date**: April 12, 2026

### Changes Added

| File | Change |
|------|--------|
| `config/settings.py` | Added dynamic threshold settings |
| `src/trading_engine.py` | Added `_get_dynamic_threshold()` method |

### New Settings

```python
USE_DYNAMIC_THRESHOLD: bool = True           # Enable volatility-based threshold
DEFAULT_CONFIDENCE_THRESHOLD: float = 0.60    # Base threshold (60-65%)
HIGH_VOL_THRESHOLD: float = 0.70            # Higher threshold in high volatility
LOW_VOL_THRESHOLD: float = 0.55             # Lower threshold in low volatility
VOLATILITY_HIGH_RATIO: float = 1.5          # vol_ratio threshold for high volatility
```

### Threshold Logic

| Condition | Threshold | Rationale |
|-----------|-----------|-----------|
| vol_ratio > 1.5 (high vol) | 70% | More certainty needed |
| vol_ratio < 0.7 (low vol) | 55% | More aggressive |
| Normal | 60% | Default |

### Notes

- Trading engine uses regime + threshold to decide trades
- Downtrend blocks all buys regardless of confidence
- Neutral requires 85% or threshold+10% (whichever higher)
- Uptrend uses dynamic threshold based on volatility
