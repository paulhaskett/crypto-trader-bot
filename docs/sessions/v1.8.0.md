# Session: v1.8.0 - Dual Training (USD + GBP) & Model Improvements
**Date**: April 10, 2026

## Overview

Fixed the model accuracy issue by implementing dual training on both USD and GBP pairs. The system now trains on both currencies and uses the appropriate model when generating signals for GBP trading.

## Problem

The investigation revealed several issues causing low model accuracy:

1. **Feature Mismatch**: When trading `BTC-GBP`, the system was:
   - Using `BTC-USD` model (correct)
   - But fetching `BTC-USD` data for features (wrong!)
   
2. **USD-only Training**: Models were trained only on USD pairs, but trading happened on GBP pairs with different price scales and volatility.

3. **Low Accuracy**: Models showed ~45-60% accuracy, barely above random.

## Root Cause Analysis

| Issue | Impact |
|-------|--------|
| `predict()` mapped product_id to USD | Features fetched for USD, not GBP |
| `get_latest_features()` used mapped product_id | GBP trading got USD features = mismatch |
| Model trained on USD, features from GBP | Different scales = poor predictions |

## Changes Made

### 1. config/settings.py (Line 46-49)

```python
# Before: USD only
TRAINING_PAIRS: list = [p.replace('-GBP', '-USD') for p in get_active_pairs()]

# After: Both USD and GBP
TRAINING_PAIRS: list = get_active_pairs() + [p.replace('-GBP', '-USD') for p in get_active_pairs()]
```

Now trains on 12 pairs (6 GBP + 6 USD).

### 2. config/settings.py (Line 215)

```python
# Before: 3% outlier threshold
MAX_PRICE_DEVIATION: float = 0.03

# After: 2% outlier threshold (per model_improvements.md)
MAX_PRICE_DEVIATION: float = 0.02
```

### 3. src/ai_model.py - predict() method (Lines 1262-1298)

Fixed to prefer GBP model for GBP trading:

```python
# Before: Always mapped to USD
if product_id in self.gbp_to_usd_map:
    product_id = usd_product_id  # Used USD for BOTH model AND features

# After: Prefer GBP model, use GBP features
if product_id in self.gbp_to_usd_map:
    if product_id in self.models:
        # Use GBP model with GBP features
        model_product_id = product_id
        features_product_id = product_id
    else:
        # Fall back to USD model but STILL use GBP features
        model_product_id = usd_product_id
        features_product_id = product_id  # Use original!
```

### 4. src/ai_model.py - get_signal() method (Lines 1511-1526)

Simplified to pass original product_id to predict():

```python
# Before: Mapped to USD before calling predict()
if product_id in self.gbp_to_usd_map:
    product_id = usd_product_id
prediction = self.predict(product_id)

# After: Pass original, let predict() decide
prediction = self.predict(original_product_id)
```
