## v3.1 - AI Training Pipeline Overhaul

**Date**: May 17, 2026

### Summary

Comprehensive overhaul of the AI training pipeline to improve prediction accuracy, precision, F1 score, and ultimately profit. Implemented in 4 phases following guidelines in `model_improvements.md` and `metrics_overview.md`.

### Backup

Created backup before changes: `backup/v3_20260517_103250/`

### Phase 1: Class Weight Balancing + HistGradientBoostingClassifier

**Purpose**: Address class imbalance and improve training speed.

**Changes** (`src/ai/training.py`):
- Added `class_weight='balanced_subsample'` to RandomForestClassifier
- Replaced GradientBoostingClassifier with HistGradientBoostingClassifier (faster + built-in class balancing)

**Result**: Better handling of imbalanced BUY/HOLD/SELL labels.

### Phase 2: Hyperparameter Tuning + Feature Selection + Data Window

**Purpose**: Find optimal model parameters and use more data for signals.

**Changes** (`src/ai/training.py`, `src/ai/signals.py`):
- Added GridSearchCV for RF hyperparameter optimization
- Added feature importance logging after training
- Added cross-validation F1 scoring
- Increased signal generation data window from 7 to 14 days

**Result**: Better-tuned models, more informative features, more market context for signals.

### Phase 3: Dynamic Weighting + Regime-Aware Ensemble

**Purpose**: Make ensemble smarter about how models vote.

**Changes** (`src/ai/ensemble.py`):
- Enhanced `_get_action()` with agreement-based confidence scoring:
  - Unanimous (all 4 models agree): +15% confidence boost
  - Split (2-2): -30% confidence reduction
- Added `_get_agreement_level()` method: unanimous/majority/split
- Enhanced regime adjustments:
  - Uptrend + BUY: +15% boost
  - Downtrend + BUY: -30% reduction
  - Downtrend + SELL: +15% boost
- Added `agreement_level` field to signal output

**Result**: Higher confidence when models agree, lower when split.

### Phase 4: Metrics Tracking + Dashboard Updates

**Purpose**: Track precision per signal type (BUY vs SELL) as required by `metrics_overview.md`.

**Changes**:

**API** (`src/api_worker.py`):
- Added `signal_precision` calculation to `/api/performance`:
  - BUY trades: total, wins, losses, precision, total_pnl
  - SELL trades: total, wins, losses, precision, total_pnl
- Only shows precision when ≥5 trades and ≥30 total trades (min threshold)

**Dashboard** (`src/templates/performance.html`):
- Added two new cards:
  - **BUY Signal Precision**: trades, wins, losses, precision %, P&L
  - **SELL Signal Precision**: trades, wins, losses, precision %, P&L
- Color-coded: green when ≥50%, red when <50%
- Precision shows "---" when insufficient data

### Configuration

New settings in `config/settings.py`:
```python
ENSEMBLE_VOTE_THRESHOLD: float = 0.75  # 75% = 3/4 models must agree
MODEL_CONFIDENCE_THRESHOLD: float = 0.65  # Minimum probability
USE_RIDGE_MODEL: bool = True  # Ridge classifier enabled
RIDGE_ALPHA: float = 1.0  # Regularization strength
```
