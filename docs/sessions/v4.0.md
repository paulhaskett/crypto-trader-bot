# Session: v4.0 - Models Page Bugs, Auto-Retrain Fix & API Consolidation

**Date**: May 31, 2026

## Summary

Fixed two models page bugs (agreement matrix N/A, auto-retrain not working), removed duplicate API endpoints, and consolidated retrain status into a single endpoint.

## Bugs Fixed

### 1. Agreement Matrix Shows N/A

**Root Cause**: `EnsemblePredictor.predict()` returned `{action, confidence, prediction, agreement, ...}` but did NOT include individual model predictions. The signal cache was written without `rf_prediction`, `lr_prediction`, etc. — frontend read `None` and displayed `'N/A'`.

**Fix** (`src/ai/signals.py:98-106`): Added individual predictions + confidences to signal result:
```python
result['rf_prediction'] = predictions.get('rf')
result['lr_prediction'] = predictions.get('lr')
...  # all 5 model types
for model_type in ['rf', 'lr', 'mlp', 'gb', 'ridge']:
    p = probas.get(model_type, [0.33, 0.34, 0.33])
    result[f'{model_type}_confidence'] = max(p)
```

### 2. Auto-Retrain Not Working / Background Signal Refresh Silent Failure

**Root Cause**: Two methods missing from the new modular `AIModel` class:
- `scheduled_retrain()` — called by APScheduler for automatic weekly retraining
- `write_signals_to_file()` — called in `refresh_signals_background()` for periodic signal refresh

Both calls were wrapped in try/except, so they failed silently.

**Fix** (`src/ai/__init__.py`): Added both methods to `AIModel`:
- `scheduled_retrain()` — calls `retrain_all_models(force=True)` then `update_last_retrain_date()`
- `write_signals_to_file(signals)` — wraps `write_signal_cache(signals)` from cache_manager
